# Phase 5.3: LoRA Training Execution & Epoch Tuning

This guide provides the exact bash commands and scripts to execute the LoRA training for all three models (Z-Image, Flux, SDXL) inside the GPU container.

---

## 1. Environment Configurations
Before launching the training scripts, ensure that your training environment variables are set up to use the GPUs properly:

```bash
# Set CUDA device visibility (e.g. use GPU 0)
export CUDA_VISIBLE_DEVICES=0

# Configure HuggingFace/Modelscope caching paths (optional)
export HF_HOME=/home/project_id_10/.cache/huggingface
```

---

## 2. Model 1: Z-Image LoRA Training
Z-Image uses a custom DiT architecture. Run the following command from the `/home/project_id_10/Captioning/DiffSynth-Studio/` directory:

```bash
accelerate launch examples/z_image/model_training/train.py \
  --dataset_base_path ../data/id10/zimage \
  --dataset_metadata_path ../data/id10/zimage/metadata_zimage.jsonl \
  --max_pixels 1048576 \
  --dataset_repeat 50 \
  --model_id_with_origin_paths "Tongyi-MAI/Z-Image:transformer/*.safetensors,Tongyi-MAI/Z-Image-Turbo:text_encoder/*.safetensors,Tongyi-MAI/Z-Image-Turbo:vae/diffusion_pytorch_model.safetensors" \
  --learning_rate 1e-4 \
  --num_epochs 5 \
  --remove_prefix_in_ckpt "pipe.dit." \
  --output_path "../data/id10/zimage/models/Z-Image_lora" \
  --lora_base_model "dit" \
  --lora_target_modules "to_q,to_k,to_v,to_out.0,w1,w2,w3" \
  --lora_rank 16 \
  --use_gradient_checkpointing \
  --dataset_num_workers 4
```

* **lora_rank**: Set to `16` (or `32` for deeper capacity).
* **dataset_repeat**: Set to `50` to repeat the dataset inside each epoch for stable training steps.

---

## 3. Model 2: FLUX.1-dev LoRA Training
FLUX.1-dev uses a double-stream block architecture. Run the following command from the `/home/project_id_10/Captioning/DiffSynth-Studio/` directory:

```bash
accelerate launch examples/flux/model_training/train.py \
  --dataset_base_path ../data/id10/flux \
  --dataset_metadata_path ../data/id10/flux/metadata_flux.jsonl \
  --max_pixels 1048576 \
  --dataset_repeat 20 \
  --model_id_with_origin_paths "black-forest-labs/FLUX.1-dev:flux1-dev.safetensors,black-forest-labs/FLUX.1-dev:text_encoder/model.safetensors,black-forest-labs/FLUX.1-dev:text_encoder_2/*.safetensors,black-forest-labs/FLUX.1-dev:ae.safetensors" \
  --learning_rate 1e-4 \
  --num_epochs 5 \
  --remove_prefix_in_ckpt "pipe.dit." \
  --output_path "../data/id10/flux/models/FLUX.1-dev_lora" \
  --lora_base_model "dit" \
  --lora_target_modules "a_to_qkv,b_to_qkv,ff_a.0,ff_a.2,ff_b.0,ff_b.2,a_to_out,b_to_out,proj_out,norm.linear,norm1_a.linear,norm1_b.linear,to_qkv_mlp" \
  --lora_rank 16 \
  --align_to_opensource_format \
  --use_gradient_checkpointing \
  --dataset_num_workers 4
```

* **align_to_opensource_format**: Automatically converts model weights to the standard ComfyUI/webUI-compatible format.

## 4. Model 3: SDXL LoRA Training
SDXL uses the traditional UNet cross-attention layers. Run the following command from the `/home/project_id_10/Captioning/DiffSynth-Studio/` directory:

```bash
accelerate launch examples/stable_diffusion_xl/model_training/train.py \
  --dataset_base_path ../data/id10/sdxl \
  --dataset_metadata_path ../data/id10/sdxl/metadata_sdxl.jsonl \
  --height 1024 \
  --width 1024 \
  --dataset_repeat 50 \
  --model_id_with_origin_paths "stabilityai/stable-diffusion-xl-base-1.0:text_encoder/model.safetensors,stabilityai/stable-diffusion-xl-base-1.0:text_encoder_2/model.safetensors,stabilityai/stable-diffusion-xl-base-1.0:unet/diffusion_pytorch_model.safetensors,stabilityai/stable-diffusion-xl-base-1.0:vae/diffusion_pytorch_model.safetensors" \
  --learning_rate 1e-4 \
  --num_epochs 5 \
  --remove_prefix_in_ckpt "pipe.unet." \
  --output_path "../data/id10/sdxl/models/stable-diffusion-xl-base-1.0_lora" \
  --lora_base_model "unet" \
  --lora_target_modules "to_q,to_k,to_v,to_out.0" \
  --lora_rank 16 \
  --use_gradient_checkpointing \
  --dataset_num_workers 4
```

---

## 5. Monitoring & Saving Checkpoints
* **Gradient Accumulation**: If you encounter Out-of-Memory (OOM) errors, add `--gradient_accumulation_steps 2` or `4` to reduce effective batch size.
* **Tuning Learning Rate**: Start with `1e-4`. If style is not bleeding in, try `2e-4`. If patterns become unstable, lower it to `5e-5`.
* **Saves**: The outputs will be saved in `data/id10/{model_name}/models/` containing the `.safetensors` LoRA files at each epoch.
