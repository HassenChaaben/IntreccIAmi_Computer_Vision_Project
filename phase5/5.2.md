## 2. Crucial Step: Metadata Key Compatibility Formatting

> [!WARNING]
> DiffSynth-Studio's training scripts (`train.py`) expect a **`"prompt"`** key in the metadata dictionary to extract the text description (e.g. `data["prompt"]`). 
> Our generated JSONL files currently use the key **`"caption"`** (e.g. `{"image": "...", "caption": "..."}`). Running training directly will result in a `KeyError: 'prompt'` crash.

To make the metadata compatible, run the following Python script once from the workspace root to convert the `"caption"` key to `"prompt"` in all three `.jsonl` files:

```bash
python -c "
import json, os

workspace = '/home/project_id_10/DiffSynth-Studio'
models = ['zimage', 'flux', 'sdxl']

for model in models:
    jsonl_path = os.path.join(workspace, 'data', 'id10', model, f'metadata_{model}.jsonl')
    if not os.path.exists(jsonl_path):
        print(f'Skipping {model} (Not found)')
        continue
        
    print(f'Converting {model} metadata keys...')
    updated_lines = []
    with open(jsonl_path, 'r', encoding='utf-8') as f:
        for line in f:
            if not line.strip():
                continue
            item = json.loads(line)
            if 'caption' in item:
                item['prompt'] = item.pop('caption')
            updated_lines.append(json.dumps(item) + '\n')
            
    with open(jsonl_path, 'w', encoding='utf-8') as f:
        f.writelines(updated_lines)
    print(f'Successfully updated {jsonl_path}')
"
```

This will convert your JSONL format to:
`{"image": "images/IMG_XXXX.jpg", "prompt": "intrecciami-style: ..."}`

## 3. Path Resolution Verification
When launching training:
* `--dataset_base_path` must point to `/home/project_id_10/DiffSynth-Studio/data/id10/{model_name}/`
* The `image` path inside the JSONL (e.g., `images/IMG_6865.jpg`) will be appended to `--dataset_base_path` automatically by DiffSynth-Studio (`ToAbsolutePath` operator), resolving to the correct absolute image paths on the server.
