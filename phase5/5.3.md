# Phase 5.3: LoRA Training Execution & Epoch Tuning

This guide provides the exact bash commands used to execute the LoRA training for Z-Image and FLUX inside the GPU container (`cvdl2026-gr18-GPU1`).

---

## 1. Environment Configuration

Before launching the training scripts, set up the environment:

```bash
# Set download source to Hugging Face (CRITICAL: prevents slow ModelScope downloads)
export DIFFSYNTH_DOWNLOAD_SOURCE=huggingface

# Configure HuggingFace caching paths (optional)
export HF_HOME=/home/project_id_10/.cache/huggingface
```

---

## 2. Model 1: Z-Image LoRA Training

Z-Image uses a custom DiT architecture. Training was executed on a single GPU:

```bash
cd /home/project_id_10/DiffSynth-Studio

CUDA_VISIBLE_DEVICES=0 accelerate launch --num_processes 1 \
  examples/z_image/model_training/train.py \
  --dataset_base_path /home/project_id_10/DiffSynth-Studio/data/id10/zimage \
  --dataset_metadata_path /home/project_id_10/DiffSynth-Studio/data/id10/zimage/metadata_zimage.jsonl \
  --tokenizer_path /home/project_id_10/DiffSynth-Studio/models/Tongyi-MAI/Z-Image-Turbo/tokenizer \
  --max_pixels 1048576 \
  --dataset_repeat 50 \
  --model_id_with_origin_paths "Tongyi-MAI/Z-Image:transformer/*.safetensors,Tongyi-MAI/Z-Image-Turbo:text_encoder/*.safetensors,Tongyi-MAI/Z-Image-Turbo:vae/diffusion_pytorch_model.safetensors" \
  --learning_rate 1e-4 \
  --num_epochs 5 \
  --remove_prefix_in_ckpt "pipe.dit." \
  --output_path "/home/project_id_10/DiffSynth-Studio/data/id10/zimage/models/Z-Image_lora" \
  --lora_base_model "dit" \
  --lora_target_modules "to_q,to_k,to_v,to_out.0,w1,w2,w3" \
  --lora_rank 16 \
  --use_gradient_checkpointing \
  --dataset_num_workers 4
```

**Result**: 1 epoch completed → `epoch-0.safetensors`

## 3. Model 2: FLUX.1-dev LoRA Training

FLUX uses a double-stream block architecture. Training was executed on **2 GPUs** (GPU 1 + GPU 2) using `accelerate` multi-GPU.

### 3.1 Multi-GPU Setup with `accelerate config`

Before training, configure accelerate for multi-GPU:

```bash
accelerate config
```

Answer the prompts:

| Prompt | Answer |
|--------|--------|
| In which compute environment are you running? | **This machine** |
| Which type of machine are you using? | **multi-GPU** |
| How many different machines will you use? | **1** |
| Should distributed operations be checked for errors? | **no** |
| Do you wish to optimize with torch dynamo? | **no** |
| Do you want to use DeepSpeed? | **no** |
| Do you want to use FullyShardedDataParallel? | **no** |
| Do you want to use Megatron-LM? | **no** |
| How many GPUs should be used? | **2** |
| What GPU(s) by id? | **1,2** |
| Do you wish to use FP16 or BF16? | **bf16** |

> [!WARNING]
> Set **machines = 1** (single server with 2 GPUs), NOT 2. Setting machines=2 triggers multi-node setup asking for IP/port which is wrong.

### 3.3 Epoch 1: Resumed Training from Epoch 0

Before resuming, backup epoch-0:
```bash
cp /home/project_id_10/DiffSynth-Studio/data/id10/flux/models/Flux_lora/epoch-0.safetensors \
```

```bash
  --learning_rate 1e-4 \
  --num_epochs 5 \
  --remove_prefix_in_ckpt "pipe.dit." \
  --output_path "/home/project_id_10/DiffSynth-Studio/data/id10/flux/models/Flux_lora" \
  --lora_base_model "dit" \
```

**Result**: `epoch-1.safetensors` (note: DiffSynth overwrites `epoch-0.safetensors` — that's why we backed it up first)

## 5. Monitoring & Troubleshooting

* **Gradient Accumulation**: If you encounter Out-of-Memory (OOM) errors, add `--gradient_accumulation_steps 2` or `4` to reduce effective batch size.
* **Tuning Learning Rate**: Start with `1e-4`. If style is not bleeding in, try `2e-4`. If patterns become unstable, lower it to `5e-5`.
* **Saves**: The outputs are saved in `data/id10/{model_name}/models/` containing the `.safetensors` LoRA files at each epoch.
* **Speed**: With `--max_pixels 1048576` and `--dataset_repeat 20` on FLUX with 2 GPUs, expect ~4s/step. Each epoch ≈ 2-4 hours.
