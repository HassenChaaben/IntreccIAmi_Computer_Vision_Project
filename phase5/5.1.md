This guide explains how to prepare your environment and verify/download the required base models inside the university GPU container `cvdl2026-gr18-GPU1`.

## 2. Install/Verify DiffSynth-Studio
The `DiffSynth-Studio` folder is already part of the project repository. Install it in editable mode inside the Python environment to register all custom pipelines (Z-Image, Flux, SDXL):

```bash
# 1. Activate your conda/virtual environment (if any)
# e.g., conda activate cvdl2026

# 2. Install DiffSynth-Studio in editable mode
cd DiffSynth-Studio
pip install -e .
cd ..
```

Verify that the installation was successful and dependencies are satisfied:
```bash
python -c "import diffsynth; print('DiffSynth-Studio version:', diffsynth.__version__)"
```

## 3. Verify base models presence or download them
DiffSynth-Studio uses standard Hugging Face/Modelscope repositories to download base models. If you are running on a server without direct internet or want to pre-download them, you can use the following methods:

### 3.1 Z-Image Models
Z-Image uses the transformer model from `Tongyi-MAI/Z-Image` and the text encoder/VAE from `Tongyi-MAI/Z-Image-Turbo`.
* **Model IDs**: 
  - `Tongyi-MAI/Z-Image`
  - `Tongyi-MAI/Z-Image-Turbo`

Verify download path:
```python
# Run inside python shell to pre-fetch and check:
from huggingface_hub import snapshot_download
snapshot_download(repo_id="Tongyi-MAI/Z-Image")
snapshot_download(repo_id="Tongyi-MAI/Z-Image-Turbo")
```

### 3.2 FLUX.1-dev Models
Flux requires the DiT transformer model, the CLIP/T5 text encoders, and the VAE model.
* **Model ID**: `black-forest-labs/FLUX.1-dev`

```python
snapshot_download(repo_id="black-forest-labs/FLUX.1-dev")
```

### 3.3 SDXL Base Model
SDXL requires the base model, UNet, and VAE.
* **Model ID**: `stabilityai/stable-diffusion-xl-base-1.0`

```python
snapshot_download(repo_id="stabilityai/stable-diffusion-xl-base-1.0")
```

> [!NOTE]
> On the university GPU server, Hugging Face models are cached in `~/.cache/huggingface/hub/`. If you encounter rate limits or internet restrictions, make sure to set the environment variable:
> `export HF_ENDPOINT=https://hf-mirror.com` (if downloading from China-based servers is faster) or copy cached models from the shared group directories.
