# Phase 5.4: Post-Training Inference Validation

This guide explains how to perform inference using the trained LoRA models and compare the results with the pre-training baseline.

---

## 1. How LoRA Loading Works in DiffSynth-Studio

DiffSynth-Studio's pipeline inherits from `BasePipeline` which includes a standard `load_lora` method to patch the base models.

```python
# For Z-Image (DiT architecture):
pipe.load_lora(module=pipe.dit, lora_config="path/to/Z-Image_lora/epoch-0.safetensors")

# For Flux (DiT architecture):
pipe.load_lora(pipe.dit, "path/to/Flux_lora/epoch-0.safetensors", alpha=1)
```

> [!NOTE]
> Z-Image uses keyword arguments (`module=`, `lora_config=`), while FLUX uses positional arguments (`pipe.dit, path, alpha=1`).

## 2. Inference Scripts Used

All scripts are located in `phase5/` and designed to run on the GPU server inside the DiffSynth-Studio directory. Each script:
- Has a `--device` flag (default `cuda`)
- Has `--start_index` / `--end_index` flags for partial runs
- Skips already-generated images (resume-friendly)
- Saves outputs to `data/id10/{model}/models/{Model}_lora/before-after_results/`

### 2.1 Z-Image Inference Scripts

#### `z_score_generate_unseen.py` — Z-Image Before/After on 10 Unseen Prompts

Generates **before** images (base model) and **after** images (LoRA fine-tuned) for 10 hand-crafted unseen prompts covering all weave techniques.

```bash
cd /home/project_id_10/DiffSynth-Studio
CUDA_VISIBLE_DEVICES=2 python z_score_generate_unseen.py --epoch 0
```

- Output: `before_unseen_{1..10}.png` and `after_unseen_{1..10}.png`
- Location: `data/id10/zimage/models/Z-Image_lora/before-after_results/`

#### `z_score_generate_seen.py` — Z-Image Before/After on 10 Seen Prompts

Generates **before** (base model) and **after** (LoRA) images for 10 training prompts.

```bash
CUDA_VISIBLE_DEVICES=2 python z_score_generate_seen.py --epoch 0
```

- Output: `before_seen_{1..10}.png` and `after_seen_{1..10}.png`
- Location: `data/id10/zimage/models/Z-Image_lora/before-after_results/`

### 2.2 FLUX Inference Scripts

## 4. EOF-Style Commands to Run Scripts on the GPU Server

To create and run any script on the GPU server, use this pattern:

```bash
# Copy script from Windows to the server
scp phase5/flux_generate_unseen.py cvdl2026-gr18@192.168.80.138:/disks/disk3/cvdl2026-gr18/

# SSH into the server
ssh cvdl2026-gr18@192.168.80.138

# Copy into the Docker container
docker cp /disks/disk3/cvdl2026-gr18/flux_generate_unseen.py cvdl2026-gr18-GPU1:/home/project_id_10/DiffSynth-Studio/

# Enter the container
docker exec -it cvdl2026-gr18-GPU1 bash

# Run
cd /home/project_id_10/DiffSynth-Studio
CUDA_VISIBLE_DEVICES=2 python flux_generate_unseen.py --epoch 0
```

Or create scripts inline with heredoc:

```bash
docker exec -i cvdl2026-gr18-GPU1 bash << 'EOF'
cd /home/project_id_10/DiffSynth-Studio
CUDA_VISIBLE_DEVICES=2 python flux_generate_unseen.py --epoch 0
EOF
```
