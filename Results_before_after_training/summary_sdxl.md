# SDXL LoRA Results & Training Analysis

This document provides a comprehensive visual and technical analysis of all generated results for the **Stable Diffusion XL (SDXL)** model. It details the training performance and the progressive visual improvements observed across Epochs 0–3, with both seen (10) and unseen (10) prompt evaluations at each checkpoint.

> All result folders live under: `Results_before_after_training/SDXL/`

---

## 1. Hyperparameters & Setup

| Parameter | Value |
| :--- | :--- |
| **Base Model** | `stabilityai/stable-diffusion-xl-base-1.0` |
| **LoRA Rank / Alpha** | 32 / 32 |
| **Target Modules** | UNet cross-attention & projection weights |
| **Hardware** | Multi-GPU (`CUDA_VISIBLE_DEVICES=1,2`), distributed via `accelerate` |
| **Learning Rate** | `1e-4` |
| **Precision** | `float32` for VAE decoding (required — see note below) |
| **dataset_repeat** | 20 (→ 177 × 20 = 3,540 instances/epoch) |
| **Dataset Size** | 177 training images |
| **Epochs Completed** | 0, 1, 2, 3 (14,160 cumulative instances at Epoch 3) |

> [!WARNING]
> **VAE float16 Instability:** The SDXL base VAE generates solid black images when run in float16 due to numerical underflow in the decoder. All inference requires `torch.float32` decoding to avoid this.

## 3. Epoch 0 — Early Optimization Phase (3,540 instances)

**Status:** The model establishes basic cross-attention mapping for `intrecciami-style`. Macro layout converges but fine details (stitching, borders) are lost in the VAE $8\times$ downsampling bottleneck. Woven grids show low-frequency waviness (spectral bias). Color bleeding at contrasting crossing points.
